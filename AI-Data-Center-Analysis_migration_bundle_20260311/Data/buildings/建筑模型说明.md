# DRL_DC 建筑模型说明 (清理后)

本文档描述清理后的 `DRL_DC_training.epJSON` 和 `DRL_DC_evaluation.epJSON` 模型。

---

## 1. PlantLoop 拓扑

模型包含 **2 个水环路**:

### Chilled Water Loop (冷冻水环路)
- **供给侧**: Chiller (离心式冷水机组 1230tons) + Waterside Economizer (水侧自然冷却换热器) + Bypass
- **需求侧**: CRAH Cooling Coil (机房空调冷却盘管) + Secondary Pump (变频二次泵) + Bypass
- **控制**: CommonPipe, SingleSetpoint, Optimal load distribution
- **温度范围**: 1-40 C

### Condenser Water Loop (冷却水环路)
- **供给侧**: Cooling Tower (冷却塔) + Constant Speed Pump + Bypass
- **需求侧**: Chiller condenser side + Waterside Economizer + Bypass
- **控制**: SingleSetpoint, Optimal load distribution
- **温度范围**: 5-80 C

---

## 2. EMS Programs (剩余 2 个)

### P_1 -- HVAC 控制程序
根据 DRL agent 输出的归一化动作值，映射到实际设备控制值:
- **CRAH_T**: 送风温度 (11-22 C)，带 +/-0.5 C 步进约束
- **CRAH_Fan**: 风机风量 (500-2000 kg/s)，带 +/-100 步进约束
- **Chiller_T**: 冷冻水设定温度 (5.6-10 C)，带 +/-0.5 C 步进约束
- **CT_Pump**: 冷却水泵流量 (300-5000 kg/s)，带 +/-100 步进约束
- **安全保护**: 当机房温度 > 30 C 时，强制最大冷却

### P_2 -- IT 负载控制程序
- **Training 版**: 随机游走 (RandomUniform)，ITE_rate 在 0.25-0.75 之间波动，模拟随机 IT 负载
- **Evaluation 版**: `SET ITE_rate = ITE_S`，直接使用 agent 动作值

## 8. 已删除内容及原因

### 删除的顶层对象类型
| 类型 | 子对象 | 原因 |
|---|---|---|
| ElectricLoadCenter:Distribution | Load Center | 电力负载中心 -- 无 PV/Wind/Battery 后不需要 |
| ElectricLoadCenter:Generators | Generator List | 发电机列表 -- 同上 |
| ElectricLoadCenter:Inverter:PVWatts | PV Inverter | PV 逆变器 -- PV 已移除 |
| ElectricLoadCenter:Storage:LiIonNMCBattery | LiIonBattery | 锂电池储能 -- 当前阶段不使用 |
| Generator:PVWatts | PV_farm | 光伏发电 -- 当前阶段不使用 |
| Generator:WindTurbine | WT1, WT2, WT3 | 风力发电 -- 当前阶段不使用 |

### 删除的 EMS 对象
- **Actuators**: Charge_rate, Discharge_rate, min_v, max_v (电池充放电控制)
- **Sensors**: SoC, B_V, Charge_S, Charge_Def, Discharge_S (电池状态); Wind_Pro1/2/3, Solar_Pro (发电量); Pro, Dem (总发电/需求)
- **Programs**: P_3 (电池充放电逻辑), P_4 (可再生能源 vs 需求判断)
- **ProgramCallingManager**: P4 整体删除; P1 中移除了 P_3 的调用
- **GlobalVariable**: min_value, max_value (P_3 电池控制用临时变量)

### 删除的 Schedule:Constant
- Charge_Def_Set, Charge_Set, Discharge_Set (电池动作)
- min_v, max_v (P_4 可再生能源输出)

### 删除的 Output:Variable
- Output:Variable 16/17/18: LiIonBattery 的 Charge State, Charge Power, Discharge Power

**删除原因总结**: 当前阶段聚焦于 HVAC + IT 负载联合优化，PV/Wind/Battery 功能暂不使用。移除这些对象可简化模型、减少仿真时间、避免未连接组件导致的 EnergyPlus 警告。
