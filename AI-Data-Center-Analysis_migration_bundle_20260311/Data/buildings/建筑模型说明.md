# DRL_DC 建筑模型说明 (清理后)

本文档描述清理后的 `DRL_DC_training.epJSON` 和 `DRL_DC_evaluation.epJSON` 模型。

---

## 1. PlantLoop 拓扑

模型包含 **2 个水环路**:

### Chilled Water Loop (冷冻水环路)
- **供给侧**: Chiller (离心式冷水机组 1230tons) + Waterside Economizer (水侧自然冷却换热器) + TES Tank (Use Side, 放冷) + Bypass
- **需求侧**: CRAH Cooling Coil (机房空调冷却盘管) + Secondary Pump (变频二次泵) + TES Tank (Source Side, 充冷) + Bypass
- **控制**: CommonPipe, SingleSetpoint, Optimal load distribution
- **温度范围**: 1-40 C

**M1 新增: TES (蓄冷水罐) 单 loop 并联接入** — 见第 9 节。

### Condenser Water Loop (冷却水环路)
- **供给侧**: Cooling Tower (冷却塔) + Constant Speed Pump + Bypass
- **需求侧**: Chiller condenser side + Waterside Economizer + Bypass
- **控制**: SingleSetpoint, Optimal load distribution
- **温度范围**: 5-80 C

## 2. EMS Programs (剩余 2 个)

### P_1 -- HVAC 控制程序 (12 行，纯滑动窗口)
根据 DRL agent 输出的归一化动作值，映射到实际设备控制值:
- **CRAH_T**: 送风温度 (11-22 C)，带 +/-0.5 C 步进约束
- **CRAH_Fan**: 风机风量 (500-2000 kg/s)，带 +/-100 步进约束
- **Chiller_T**: 冷冻水设定温度 (5.6-10 C)，带 +/-0.5 C 步进约束
- **CT_Pump**: 冷却水泵流量 (300-3300 kg/s)，带 +/-100 步进约束
- **无安全保护**: E0.1 修复中删除了 >30C 紧急保护（原因：41.8MW 过冷导致 1 步降温 15°C，比超温更有害）

### P_2 -- IT 负载控制程序
- **Training 版**: 随机游走 (RandomUniform)，ITE_rate 在 0.05-0.80 之间波动，模拟随机 IT 负载
- **Evaluation 版**: `SET ITE_rate = ITE_S`，直接使用 agent 动作值

### 9.2 接入拓扑 (方案 A — 单 Chilled Water Loop 并联)

```
Supply 侧 (冷源):
  Inlet Branch → Splitter → {Branch 1: Chiller,
                              Branch 2: Chiller Bypass,
                              Branch 3: TES Use Side (放冷)} → Mixer → Outlet Branch

Demand 侧 (冷汇):
  Inlet Branch → Splitter → {Branch 1: Coil Bypass,
                              Branch 2: CRAH Coil,
                              Branch 3: TES Source Side (充冷),
                              Bypass Branch} → Mixer → Outlet Branch
```

- **TES Use Side** 挂在 Supply Branch 3，作为制冷**供应方**（放冷给回路）
- **TES Source Side** 挂在 Demand Branch 3，作为制冷**消耗方**（从 chiller 充冷）
- 不添加独立泵 — 复用已有 Primary Pump (Constant) + Secondary Pump (VariableSpeed)
- TES 已加入 `Chilled Water Loop Cooling Equipment List` 作为第 3 个 equipment

### 9.3 节点命名

| 节点 | 物理含义 |
|---|---|
| `CW Tank Use Inlet Node` | TES 顶端进口 (热水回流) |
| `CW Tank Use Outlet Node` | TES 底端出口 (冷水放冷给 CRAH 侧) |
| `CW Tank Source Inlet Node` | TES 底端进口 (chiller 冷水入罐) |
| `CW Tank Source Outlet Node` | TES 顶端出口 (回流到 chiller 回路) |

### 9.4 新增 Output:Variable (编号 16/17/18/20)

| Schedule:Constant | 初值 | 用途 |
|-------------------|------|------|

**EMS 自动控制的 Schedule:Constant**（供 sinergym 观测 / 调试）：

| Schedule:Constant | 用途 |
|-------------------|------|
| `TES_Use_Avail_Sch` | EMS 写，Use 侧 availability（放冷开关）|
| `TES_Source_Avail_Sch` | EMS 写，Source 侧 availability（充冷开关）|
| `TES_SOC_Obs` | EMS 写，当前 SOC 观测值 ∈ [0, 1] |
| `Chiller_Avail_Sch` | EMS 写，Chiller 开关状态（1=正常，0=被迫关停放电）|

**EMS:Actuator**：

| Actuator 名称 | 作用对象 | 说明 |
|---------------|---------|------|
| `TES_DRL` | `TES_Set` | RL 动作入口（外部写入）|
| `TES_Use_Avail` | `TES_Use_Avail_Sch` | EMS 写 |
| `TES_Source_Avail` | `TES_Source_Avail_Sch` | EMS 写 |
| `TES_SOC_Actuator` | `TES_SOC_Obs` | EMS 写 SOC 值 |
| `Chiller_Branch_Avail` | `Chilled Water Loop Supply Branch 1 / On/Off Supervisory` | 关 Chiller 分支 |
| `Chiller_Component_Avail` | `Chiller:Electric:EIR / On/Off Supervisory` | 关 Chiller 组件 |
| `Chiller_Out_T_SP` | Chiller Supply Outlet Node 温度 Setpoint | 设极高值迫使关停 |
| `Chiller_In_MFlow_Max` | Chiller Supply Inlet Node 最大流量 | 设 0 切断流量 |
| `Chiller_Avail_Obs` | `Chiller_Avail_Sch` | EMS 写观测状态 |

**EMS:Sensor**：

| Sensor | 读取变量 |
|--------|---------|
| `T_node_1` ~ `T_node_10` | `Chilled Water Thermal Storage Temperature Node 1..10` @ `Chilled Water Tank` |
| `TES_Set_Sensor` | `Schedule Value` @ `TES_Set`（读 RL 动作）|

**EMS:Program**：

- **P_5** — TES 充放模式控制（CallingPoint: `BeginTimestepBeforePredictor`）
  - 读 TES_Set_Sensor、SOC
  - SOC 物理约束优先：SOC<0.02 禁放，SOC>0.98 禁充
  - 否则按动作：>+0.05 充电模式，<-0.05 放电模式，其他静置
  - **放电模式叠加关 Chiller**（4 个 actuator 一起设）
- **P_6** — SOC 计算（CallingPoint: `EndOfZoneTimestepAfterZoneReporting`）
  - T_tank_avg = (T_node_1 + ... + T_node_10) / 10
  - SOC = (12°C - T_tank_avg) / (12°C - 6°C)，Clip 到 [0, 1]
  - 写入 TES_SOC_Obs
- **P_7** — Chiller 关停运行时强制（CallingPoint: `InsideHVACSystemIterationLoop`）
  - 如果放电模式，在 HVAC 迭代中反复把 4 个 Chiller actuator 重设为关停，避免被内部控制器覆盖

**关键设计决策**：

1. **4 种 actuator 叠加关 Chiller**：Branch On/Off / Component On/Off / Outlet T Setpoint / Inlet MFlow Max 单独任何一个都无法彻底关停 Chiller（测试中单独用只能降到 ~1.3 MW），必须**四管齐下 + HVAC 迭代循环强制**才能真正关停
2. **TES_Set 不设 schedule_type_limits_name**：之前用 Fractional 会让 -0.5 被 E+ 裁剪为 0 进而完全失效，现在让它无限制（仅 EMS P_5 逻辑读取）
3. **P_6 的 SOC 初值**：仿真启动时 TES_SOC_Obs=0.5（schedule 默认值），首个 timestep 后写入真实 SOC。RL 第一个 obs 可能看到 0.5 是正常现象

### 9.8 M1-A3 验证结果

3 场景 7-day 南京仿真（DRL_DC_training.epJSON，15 分钟 timestep，672 steps）：

| 工况 | TES_Set | Chiller Elec | Chiller Cool | Use HT | Source HT | SOC 范围 |
|------|---------|--------------|--------------|--------|-----------|----------|

- 充电/静置不受 A3b 修改影响

### 9.10 已知限制（留给后续阶段）

- **Zone 温度过冲**：C_discharge 场景 SOC 放尽后 Chiller 恢复有 ~2 timestep 滞后，期间 Zone 温度可能冲到 ~41°C。RL 训练中会通过 comfort penalty 学到"不要在低 SOC 继续放"。如评估期过冲严重，考虑在 EMS P_5 加"SOC 预警"逻辑（SOC < 0.1 时提前恢复 Chiller）。
- **Source 充冷依赖 Chiller 有余量**：当 Chiller 已满载满足 CRAH 负载时，TES 充冷需求无法得到满足（Source HT 低）。论文讨论时应明确：TES 起"时间转移"作用，必须在 Chiller 有余量时预充。
