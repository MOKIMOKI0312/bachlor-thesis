# DRL_DC 建筑模型说明 (清理后)

本文档描述清理后的 `DRL_DC_training.epJSON` 和 `DRL_DC_evaluation.epJSON` 模型。

---

## 1. PlantLoop 拓扑

模型包含 **2 个水环路**:

### Chilled Water Loop (冷冻水环路)
- **供给侧**: Chiller (离心式冷水机组 1230tons) + Waterside Economizer (水侧自然冷却换热器) + TES Tank (Use Side, 放冷) + Bypass
- **需求侧**: CRAH Cooling Coil (机房空调冷却盘管) + Secondary Pump (变频二次泵) + TES Tank (Source Side, 充冷) + Bypass
- **控制**: CommonPipe, SingleSetpoint, Optimal load distribution
- **温度范围**: 1-40 C

**M1 新增: TES (蓄冷水罐) 单 loop 并联接入** — 见第 9 节。

### Condenser Water Loop (冷却水环路)
- **供给侧**: Cooling Tower (冷却塔) + Constant Speed Pump + Bypass
- **需求侧**: Chiller condenser side + Waterside Economizer + Bypass
- **控制**: SingleSetpoint, Optimal load distribution
- **温度范围**: 5-80 C

## 2. EMS Programs (剩余 2 个)

### P_1 -- HVAC 控制程序 (12 行，纯滑动窗口)
根据 DRL agent 输出的归一化动作值，映射到实际设备控制值:
- **CRAH_T**: 送风温度 (11-22 C)，带 +/-0.5 C 步进约束
- **CRAH_Fan**: 风机风量 (500-2000 kg/s)，带 +/-100 步进约束
- **Chiller_T**: 冷冻水设定温度 (5.6-10 C)，带 +/-0.5 C 步进约束
- **CT_Pump**: 冷却水泵流量 (300-3300 kg/s)，带 +/-100 步进约束
- **无安全保护**: E0.1 修复中删除了 >30C 紧急保护（原因：41.8MW 过冷导致 1 步降温 15°C，比超温更有害）

### P_2 -- IT 负载控制程序
- **Training 版**: 随机游走 (RandomUniform)，ITE_rate 在 0.05-0.80 之间波动，模拟随机 IT 负载
- **Evaluation 版**: `SET ITE_rate = ITE_S`，直接使用 agent 动作值

| ElectricLoadCenter:Inverter:PVWatts | PV Inverter | PV 逆变器 -- PV 已移除 |
| ElectricLoadCenter:Storage:LiIonNMCBattery | LiIonBattery | 锂电池储能 -- 当前阶段不使用 |
| Generator:PVWatts | PV_farm | 光伏发电 -- 当前阶段不使用 |
| Generator:WindTurbine | WT1, WT2, WT3 | 风力发电 -- 当前阶段不使用 |

### 删除的 EMS 对象
- **Actuators**: Charge_rate, Discharge_rate, min_v, max_v (电池充放电控制)
- **Sensors**: SoC, B_V, Charge_S, Charge_Def, Discharge_S (电池状态); Wind_Pro1/2/3, Solar_Pro (发电量); Pro, Dem (总发电/需求)
- **Programs**: P_3 (电池充放电逻辑), P_4 (可再生能源 vs 需求判断)
- **ProgramCallingManager**: P4 整体删除; P1 中移除了 P_3 的调用
- **GlobalVariable**: min_value, max_value (P_3 电池控制用临时变量)

### 删除的 Schedule:Constant
- Charge_Def_Set, Charge_Set, Discharge_Set (电池动作)
- min_v, max_v (P_4 可再生能源输出)

### 删除的 Output:Variable
- Output:Variable 16/17/18: LiIonBattery 的 Charge State, Charge Power, Discharge Power

**删除原因总结**: 当前阶段聚焦于 HVAC + IT 负载联合优化，PV/Wind/Battery 功能暂不使用。移除这些对象可简化模型、减少仿真时间、避免未连接组件导致的 EnergyPlus 警告。

| Use Inlet / Outlet Height | 21.5 m / 0.5 m (顶进热, 底出冷) |

### 9.2 接入拓扑 (方案 A — 单 Chilled Water Loop 并联)

```
Supply 侧 (冷源):
  Inlet Branch → Splitter → {Branch 1: Chiller,
                              Branch 2: Chiller Bypass,
                              Branch 3: TES Use Side (放冷)} → Mixer → Outlet Branch

Demand 侧 (冷汇):
  Inlet Branch → Splitter → {Branch 1: Coil Bypass,
                              Branch 2: CRAH Coil,
                              Branch 3: TES Source Side (充冷),
                              Bypass Branch} → Mixer → Outlet Branch
```

- **TES Use Side** 挂在 Supply Branch 3，作为制冷**供应方**（放冷给回路）
- **TES Source Side** 挂在 Demand Branch 3，作为制冷**消耗方**（从 chiller 充冷）
- 不添加独立泵 — 复用已有 Primary Pump (Constant) + Secondary Pump (VariableSpeed)
- TES 已加入 `Chilled Water Loop Cooling Equipment List` 作为第 3 个 equipment

### 9.3 节点命名

| 节点 | 物理含义 |
|---|---|
| `CW Tank Use Inlet Node` | TES 顶端进口 (热水回流) |
| `CW Tank Use Outlet Node` | TES 底端出口 (冷水放冷给 CRAH 侧) |
| `CW Tank Source Inlet Node` | TES 底端进口 (chiller 冷水入罐) |
| `CW Tank Source Outlet Node` | TES 顶端出口 (回流到 chiller 回路) |

### 9.4 新增 Output:Variable (编号 16/17/18/20)

- `Chilled Water Thermal Storage Final Tank Temperature` @ `Chilled Water Tank` — 平均罐温
- `Chilled Water Thermal Storage Use Side Heat Transfer Rate` @ `Chilled Water Tank` — 放冷率 (正 = 从罐放热)
- `Chilled Water Thermal Storage Source Side Heat Transfer Rate` @ `Chilled Water Tank` — 充冷率 (负 = 从源侧吸冷)
- `Chilled Water Thermal Storage Tank Heat Loss Rate` @ `Chilled Water Tank` — 罐外散热损失

### 9.5 M1-A1/A2 验证结果

- **JSON schema**：TES 对象通过 EnergyPlus 23.1 官方 schema 校验 ✓
- **Branch integrity**：1-day & 7-day 仿真均通过 E+ "All Branches passed integrity testing" ✓
- **1-day sim (南京天气, Jan 1)**：`severe=0, fatal=0, returncode=0`
- **7-day sim**：`severe=0, fatal=0, returncode=0`
  - Tank 温度稳定在 5.919 – 6.032 °C (靠近 setpoint 6 °C)
  - 最大 1-timestep ΔT = 0.033 °C (< 10 °C 合理阈值)
  - Source side 在暖机/瞬态期间触发充冷 (最大 −673 kW)
  - Use side 当前为 0 — **设计符合预期**：无 EMS 显式指令使 TES 放冷时，CRAH 流量走阻力更低的 Coil Bypass 路径或直接取自 chiller 供水。M1-A3 将通过 EMS 加入 Use/Source Availability Schedule 或 SetpointManager 控制放冷时机。
