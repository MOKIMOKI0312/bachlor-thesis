# Weather Data Rules

本目录保存 EnergyPlus 模型使用的天气文件。

## 当前默认天气

- 当前默认案例使用：`CHN_JS_Nanjing.582380_TMYx.2009-2023.epw`
- 该文件对应南京 TMYx EPW 天气数据。
- 当前 runner 默认使用该南京天气文件。

## 数据口径

- EPW 是 EnergyPlus 物理仿真的天气输入。
- PV 出力不直接从本目录中的 EPW 自动生成；PV 预期出力默认由 PVGIS 生成，并保存在 `../inputs/` 或 `../inputs/pvgis/`。
- 如果天气文件用于生成 PVGIS PV 出力，必须在对应 PVGIS 说明中记录天气来源或 PVGIS weather database。

## 多地点/多场景规则

- 本目录允许后续保存不同地点、不同年份或不同气象源的 EPW 文件。
- 新增天气文件时，文件名应尽量包含国家/地区、城市或站点、年份范围、数据源。
- 多地点实验必须同步维护对应关系：
  - weather 文件
  - price 文件
  - PV 文件
  - model 或 scenario 名称
- 不同地点或不同年份的数据可以用于实验假设，但必须在说明文档中明确，不要默认为真实同源数据。

## 修改规则

- 不手工编辑原始 EPW 内容；如需修正，应保留原始文件，并生成明确命名的派生文件。
- 不覆盖默认南京天气文件，除非用户明确要求。
- 不把临时下载、未校验或不确定来源的天气文件长期放在本目录。
- 新增或替换 EPW 时，必须记录来源、站点、年份范围、下载日期和用途。

## Agent 分工要求

- 涉及天气文件替换、下载、派生处理或多地点场景扩展时，默认由显式 GPT-5.5 subagent 执行。
- 主 agent 默认只做只读核对、文件清单核查、来源核查和最终验收。
- 修改后必须返回：
  - 新增/修改文件清单
  - 天气数据来源
  - 站点或城市
  - 年份范围
  - 是否与 price/PV/model 场景匹配
  - 是否需要更新 runner、说明文档或论文 LaTeX
