# Repository Documentation Rules

本目录保存仓库级文档、文献材料和项目管理材料。它不是当前 EnergyPlus 模型包的代码入口。

## 当前目录分工

- `literature/`：论文、参考文献、复现材料和相关阅读笔记。
- `project_management/`：项目进度、修改建议、任务记录、论文草稿和管理性文档。

当前 EnergyPlus 模型包的技术说明应优先写入：

```text
../Nanjing-DataCenter-TES-EnergyPlus/docs/
```

## 工作边界

- 不在本目录中开发代码、修改 epJSON 或保存 EnergyPlus 运行输出。
- 不把文献材料和项目管理材料混入当前模型包目录。
- 不把历史计划直接当成当前执行口径；执行口径以根目录 `AGENTS.md`、当前用户指令和活跃代码包状态为准。
- 如果文档内容影响当前模型、runner、输入数据或验证结论，应同步检查是否需要更新 `../Nanjing-DataCenter-TES-EnergyPlus/docs/`。

## 写作规则

- 文档应区分事实、推断和假设。
- 引用代码、模型、输入数据或运行结果时，应给出明确路径。
- 不编造文献结论、数据来源、模型参数或实验结果。
- 新增长期文档时优先用清晰主题命名；项目阶段文档可以带日期。

## Agent 分工要求

- 涉及文献整理、项目管理文档重写、路线说明、论文修改或长期记录更新时，默认由显式 GPT-5.5 subagent 执行。
- 主 agent 默认只做目标澄清、只读核对、事实校验和最终验收。
- 修改后必须返回：
  - 修改文件清单
  - 信息来源
  - 哪些内容是事实、推断或假设
  - 是否需要同步更新当前模型包文档
  - 是否需要同步更新论文 LaTeX
