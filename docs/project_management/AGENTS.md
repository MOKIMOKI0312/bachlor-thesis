# Project Management And Thesis Rules

本目录保存毕业设计路线、进度、论文草稿、引用库和管理性文档。它用于组织项目推进和论文写作，不是当前 EnergyPlus 模型包的代码入口。

## 当前目录分工

- `毕业设计技术路线及进度安排/`：项目内容管理、技术路线、进度安排和阶段性决策。
- `毕业设计论文/`：论文写作材料，例如 `thesis_draft.tex` 和 `references.bib`。

## 使用边界

- 不在本目录中开发代码、修改 epJSON、保存 EnergyPlus 输出或存放临时运行产物。
- 不把历史计划自动视为当前执行口径；当前执行口径以用户最新指令、根 `AGENTS.md` 和活跃模型包状态为准。
- 管理文档中的技术判断必须能回溯到代码、模型、输入数据、日志、文献或明确用户决策。
- 不覆盖旧进度记录，除非用户明确要求；补充内容优先新建带日期或主题的文件。

## 技术路线和进度文档规则

写入 `毕业设计技术路线及进度安排/` 的内容应区分：

- 已完成事项
- 当前决策
- 待办事项
- 风险
- 未验证假设
- 证据路径

涉及当前 EnergyPlus 模型时，应引用：

```text
../../Nanjing-DataCenter-TES-EnergyPlus/
```

涉及历史目标或旧路线时，只能引用：

```text
../../_archive/project_goals/
```

并明确标注为历史背景。

## 论文文件规则

`毕业设计论文/` 中的文件要按论文产物对待：

- 修改 `thesis_draft.tex` 前，应先确认目标章节和需要引用的证据。
- 修改技术结论时，必须检查是否有模型文件、运行日志、文献或数据支撑。
- 不编造实验结果、模型参数、文献结论或引用。
- 修改 `references.bib` 时，不随意改已有 citation key。
- 删除 BibTeX 条目前，必须确认 `thesis_draft.tex` 未引用该 key。
- 新增 citation key 后，应尽量保持命名稳定，并能追溯到 `docs/literature/` 中的文献文件或外部来源。

## 与当前模型包的同步

以下内容变化时，应检查是否需要同步当前模型包文档：

- 技术路线发生变化
- 论文中描述 EnergyPlus 模型、TES、PV、电价、冷却系统或验证结果
- 论文引用了某次 EnergyPlus 运行结果
- 管理文档改变了当前主线目标

当前模型包文档位置：

```text
../../Nanjing-DataCenter-TES-EnergyPlus/docs/
```

## Agent 分工要求

- 涉及项目计划重写、进度整理、论文草稿修改、BibTeX 维护或长期路线说明时，默认由显式 GPT-5.5 subagent 执行。
- 主 agent 默认只做目标澄清、事实核对、风险审查和最终验收。
- 文档修改 subagent 必须返回：
  - 修改文件清单
  - 主要决策或文本变化
  - 证据来源
  - 未确认事项
  - 是否影响论文引用
  - 是否需要同步更新根 `README.md`、根 `AGENTS.md` 或当前模型包文档
