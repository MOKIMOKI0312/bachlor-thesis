## 一、核心结论

当前 `mpc_v2` 不是“冷机 + TES 联合 MPC”，而是 TES-only 经济 MPC 代理模型。当前优化变量只有 TES 充放冷：

- `q_ch_tes_kw_th`
- `q_dis_tes_kw_th`

冷机没有作为显式决策变量进入优化；`facility_model.py` 用固定 `cop_charge` / `cop_discharge_equiv` 折算电功率；`room_model.py` 将 `base_cooling_kw_th = alpha_it_to_cooling * IT_load` 作为外生固定冷量；`no_tes` baseline 虽然将 TES 置零，但仍保留这部分外生冷量。

因此当前结果的边界是：

- TES 可以做电价套利和有限负荷搬移；
- TES 不能通过冷机台数组合、少开机、多满载、降低低 PLR 运行、利用外温改善 COP 来产生显著节能；
- `pv_spill_kwh = 0` 是结构性结果，因为 18 MW IT 固定负荷远大于 PV 峰值；
- `tes_mpc_perfect` 相对 `baseline_no_tes` 的总电费改善约 1.26%，更像是当前模型边界和变量设计的上限，而不是求解器失效。

本计划建议停止继续强化 TES-only 代理模型，改造为：

> 基于工程可解 MILP 的冷机 - 冷水蓄冷罐联合经济 MPC：在满足机房温度约束的前提下，利用冷机 mode 切换、TES 充放冷、电价、PV 和天气预测降低运行成本与峰值负荷。

### 5.5 冷机电功率与热功率关系

- [ ] `[未开始]` 用 Kim 型 mode-based 仿射 plant power 替换固定 COP。

- [ ] `[待确认]` 是否保留 EnergyPlus EIR 曲线作为论文标准模型但不进入第一版代码。

推荐第一版：

```math
q^{chiller}_k = \sum_m \nu_{m,k}
```

```math
Q^{min}_m z_{m,k} \le \nu_{m,k} \le Q^{max}_m z_{m,k}
```

```math
P^{plant}_k = \sum_m \left(a_m \nu_{m,k} + (c_{m,0}+c_{m,1}T_{wb,k})z_{m,k}\right)
```

显式 COP 只作为解释，不作为优化主模型：

```math
COP_{eff,m}(q,T_{wb})=\frac{q}{a_m q + c_{m,0}+c_{m,1}T_{wb}}
```

不要在第一版代码中反过来优化 COP 分式，否则会把 MILP 变为非线性问题。

EnergyPlus EIR 标准形式，作为论文标准模型或后续增强：

```math
CPLR = \frac{CoolingLoad}{AvailCoolCap}
```

```math
CEIRfT = a+bT_{cw,l}+cT_{cw,l}^2+dT_{cond}+eT_{cond}^2+fT_{cw,l}T_{cond}
```

```math
CEIRfPLR = a+b\cdot CPLR+c\cdot CPLR^2
```

后续如需工程化 COP 曲线，优先离线预计算 `P=f(q,T_wb)` 并做分段线性表。

常压差线性流量映射：

```math

```

```math
q^{dis}_k = \varepsilon_{dis}(s_k)\, c_p\, \dot m^{dis}_k \,(T_{ret,k}-T_{sup,k})
```

更一般阀门关系，可在论文中说明：

```math
\dot m \propto K_v(u)\sqrt{\Delta p}
```

```math

```

```math
|\Delta u^{dis}_k| = |u^{dis}_k-u^{dis}_{k-1}| \le \Delta u^{dis,max}
```

```math
|\Delta u^{ch}_k| = |u^{ch}_k-u^{ch}_{k-1}| \le \Delta u^{ch,max}
```

建议初始参数：

```text

```

- [ ] `[未开始]` 将电力边界改为冷站 plant power。

推荐冷站边界：

```math
G_k - S_k = P^{plant}_k + P^{aux}_k - P^{pv}_k
```

若 `P_plant` 已含冷机、泵、塔，则 `P_aux` 可略去。

非负约束：

```math
G_k \ge 0
```

```math
S_k \ge 0
```

需求峰值 epigraph：

```math
G_k \le D,\quad \forall k
```

目标中加入：

```math
c^{dem}D
```

### 5.8 约束集合

- [ ] `[未开始]` 加入全部 mode、启停、冷量分配、阀门、低 PLR 约束。

冷机模式：

```math
\sum_m z_{m,k}\le 1
```

```math
z_{m,k}-z_{m,k-1}=\delta^{on}_{m,k}-\delta^{off}_{m,k}
```

```math
\sum_{\ell=0}^{M^{on}_m-1}\delta^{on}_{m,k-\ell}\le z_{m,k}
```

```math
\sum_{\ell=0}^{M^{off}_m-1}\delta^{off}_{m,k-\ell}\le 1-z_{m,k}
```

冷量分配：

```math
q^{chiller}_k = q^{load}_k + q^{ch}_k
```

```math
q^{cool}_k = q^{load}_k + q^{dis}_k
```

```math
0\le q^{ch}_k \le \bar q^{ch} u^{ch}_k
```

```math
0\le q^{dis}_k \le \bar q^{dis} u^{dis}_k
```

```math
u^{ch}_k+u^{dis}_k\le 1
```

温度软约束：

```math
T_{min}-\epsilon^T_{k,\ell}\le T_k \le T_{max}+\epsilon^T_{k,u}
```

```math
\epsilon^T_{k,\ell},\epsilon^T_{k,u}\ge 0
```

SOC 软约束：

```math
s_{min}-\epsilon^s_{k,\ell}\le s_k \le s_{max}+\epsilon^s_{k,u}
```

```math
\epsilon^s_{k,\ell},\epsilon^s_{k,u}\ge 0
```

终端 SOC：

```math
s^{target}-\epsilon^{term}_{\ell}\le s_N \le s^{target}+\epsilon^{term}_{u}
```

```math
\epsilon^{term}_{\ell},\epsilon^{term}_{u}\ge 0
```

阀门速率：

```math
-\Delta u^{max} \le u_k-u_{k-1} \le \Delta u^{max}
```

低 PLR 惩罚变量：

```math
\xi_k \ge PLR^{pref}\,Q^{avail}_k - q^{chiller}_k
```

```math
\xi_k\ge 0
```

建议 `PLR_pref = 0.7` 或 `0.8`。

### 5.9 目标函数

- [x] `[部分完成]` 当前已有电费、PV spill、TES cycle、温度/SOC slack、terminal SOC、动作变化项。
- [ ] `[未开始]` 增加 plant power、demand peak、startup、low PLR、valve movement 等联合 MPC 项。

推荐目标：

```math
\min
\sum_{k=0}^{N-1}
\Big(
c_k G_k \Delta t
+
w_{spill} S_k \Delta t
+
w_{temp}(\epsilon^T_{k,\ell}+\epsilon^T_{k,u})
+
w_{soc}(\epsilon^s_{k,\ell}+\epsilon^s_{k,u})
+
w_{valve}(|\Delta u^{ch}_k|+|\Delta u^{dis}_k|)
+
w_{cyc}(q^{ch}_k+q^{dis}_k)\Delta t
+
w_{plr}\xi_k
+
w_{start}\sum_m \delta^{on}_{m,k}
\Big)
+
w_{dem} D
+
w_{term}(\epsilon^{term}_{\ell}+\epsilon^{term}_{u})
```

线性化要求：

- `abs(delta u)` 使用正负不等式和非负辅助变量；
- 低 PLR 使用 `xi_k`；
- mode 用二进制变量；
- 不在 MILP 中使用分式 COP、`max()`、`min()`、`if` 或非线性乘积。

权重原则：

- 温度松弛应是“大 M 级”经济惩罚；
- 0.1 degC·h 温度违约代价应明显大于同一时段所有套利收益；
- SOC 违约也应远大于单步套利收益；
- 阀门变化惩罚较小，只抑制抖动；
- cycle 惩罚不能过大，否则 TES 不会动作；
- low PLR 惩罚应与 startup cost 配合使用。
