# Technical Route And Progress Rules

本目录保存毕业设计的技术路线、项目内容管理、阶段进度和决策记录。它是项目管理区，不是代码或模型执行入口。

## 当前职责

- 记录当前毕业设计主线目标。
- 记录技术路线、阶段安排、已完成事项和待办事项。
- 记录关键决策及其依据。
- 整合从旧 `项目目标/` 迁移出来的当前有效内容。
- 为论文写作提供项目过程和方法路线依据。

## 使用边界

- 不在本目录中开发代码、修改 epJSON、保存 EnergyPlus 输出或存放临时运行产物。
- 不把旧计划直接当成当前执行口径；当前执行口径以用户最新指令、根 `AGENTS.md` 和活跃模型包状态为准。
- 不把未经验证的实验结果写成已完成结论。
- 不覆盖历史记录，除非用户明确要求；新的阶段性记录优先新建带日期或主题的 Markdown 文件。

## 推荐文档结构

本目录中的管理文档应尽量包含：

- 当前目标
- 当前主线范围
- 已完成事项
- 当前待办
- 风险和阻塞
- 关键决策
- 证据路径
- 下一步最小可执行任务

## 证据要求

涉及技术事实时，应优先引用可验证路径：

```text
../../../Nanjing-DataCenter-TES-EnergyPlus/
```

涉及历史目标、旧路线或旧任务包时，只能引用：

```text
../../../_archive/project_goals/
```

并明确标注为历史背景。

涉及论文引用或文献依据时，应引用：

```text
../../literature/
../毕业设计论文/references.bib
```

## 写作规则

- 明确区分：
  - 已完成
  - 当前决策
  - 待验证
  - 风险
  - 历史背景
- 技术路线应以最短可验证路径为优先，不为了完整流程引入无必要复杂度。
- 如当前目标已经从旧 MPC/RL 路线转为 EnergyPlus 最小模型包，应明确写出，不让旧计划误导后续 agent。
- 管理文档可以记录计划，但不能替代模型、代码、输入数据和运行日志证据。

## Agent 分工要求

- 涉及路线重写、进度整理、决策记录或长期计划更新时，默认由显式 GPT-5.5 subagent 执行。
- 主 agent 默认只做目标澄清、事实核对、风险审查和最终验收。
- 修改 subagent 必须返回：
  - 修改文件清单
  - 主要路线或进度变化
  - 证据来源
  - 未确认事项
  - 是否影响论文正文
  - 是否需要同步更新根 `README.md`、根 `AGENTS.md` 或当前模型包文档
