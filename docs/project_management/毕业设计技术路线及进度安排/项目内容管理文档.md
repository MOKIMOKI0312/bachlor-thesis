# 南京数据中心 TES-PV-TOU MILP-MPC 毕业设计项目内容管理文档

## 1. 项目定位

本项目当前按“推倒重来”的方式重新收敛到一条清晰主线：

> 数据中心冷却系统 + TES + PV + TOU 电价 -> 控制导向模型 -> MILP-MPC -> baseline/场景验证 -> 经济性、鲁棒性和适用边界分析。

毕业设计的研究对象不是泛泛的数据中心节能，也不是复现旧强化学习训练流程，而是面向南京数据中心冷却蓄冷系统，构建一个可解释、可复现、可验证的预测控制研究框架。论文应回答四个核心问题：

1. 数据中心冷却蓄冷系统如何抽象为控制导向模型？
2. TES、PV 和分时电价如何共同进入 MILP-MPC 优化问题？
3. 与 no-TES baseline 和简单规则控制相比，MPC 在哪些指标上有改善或约束优势？
4. TES-MPC 的经济收益、鲁棒性和适用边界受哪些条件限制？

### 4.2 系统边界

论文系统边界应聚焦“数据中心冷却蓄冷系统的经济型预测控制”：

- 数据中心 IT 负荷产生稳定热量。
- 冷却系统负责维持机房温度约束。
- TES 将冷量生产和冷量使用在时间上解耦。
- PV 提供本地可再生电力。
- TOU 电价提供调度套利信号。
- 电网负责补足净负荷。
- MPC 根据预测信息滚动优化 TES 充放冷、购电功率和 PV 弃电。

## 5. 模型与控制问题定义

### 5.1 外部输入

| 符号 | 含义 | 单位 | 说明 |
| --- | --- | --- | --- |
| `P_IT,k` | IT 负荷 | kW | 数据中心计算负荷 |
| `T_out,k` | 室外温度 | degC | 来自天气文件或预测 |
| `P_PV,k` | 光伏出力 | kW | 来自 PV 预测输入 |
| `c_k` | 分时电价 | CNY/kWh 或 USD/MWh | 经济调度信号 |

### 5.2 状态变量、控制变量和二进制变量

| 类型 | 符号 | 含义 | 单位 |
| --- | --- | --- | --- |
| 状态变量 | `SOC_k` | TES 荷冷状态 | - |
| 状态变量 | `T_room,k` | 机房温度 | degC |
| 控制变量 | `q_ch,k` | TES 充冷功率 | kW |
| 控制变量 | `q_dis,k` | TES 放冷功率 | kW |
| 控制变量 | `P_grid,k` | 电网购电功率 | kW |
| 控制变量 | `P_spill,k` | PV 弃电功率 | kW |
| 二进制变量 | `z_ch,k` | 是否充冷 | - |
| 二进制变量 | `z_dis,k` | 是否放冷 | - |
| 松弛变量 | `s_T,k` | 温度约束松弛 | degC |
| 松弛变量 | `s_SOC,k` | SOC 约束松弛 | - |

### 5.3 关键模型

TES SOC 动态：

```text
SOC_{k+1} =
(1 - lambda * Delta t) * SOC_k
+ eta_ch * q_ch,k * Delta t / E_TES
- q_dis,k * Delta t / (eta_dis * E_TES)
```

机房温度代理模型：

```text
T_{k+1} =
a * T_k
+ b * T_out,k
+ c * P_IT,k
- d * (q_base,k + q_dis,k)
```

PV-grid 功率平衡：

```text
P_grid,k - P_spill,k =
P_facility,k + P_TES,ch,k - P_TES,dis,k - P_PV,k
```

MPC 目标函数应同时考虑：

- 购电成本；
- PV 弃电惩罚；
- TES 过度循环惩罚；
- 温度约束松弛惩罚；
- SOC 约束松弛惩罚；
- 控制动作变化惩罚；
- 终端 SOC 偏差惩罚。

### 5.4 MILP 选择依据

本问题采用 MILP 的直接原因是 TES 不能在同一时刻同时充冷和放冷，需要用二进制变量描述充放冷模式。结合线性 SOC 动态、线性温度代理模型、线性功率平衡和线性目标函数后，控制问题可以表述为混合整数线性规划。该形式具有可解释性强、约束清楚、求解器状态明确和便于闭环复现的优点。

## 6. 实验设计

### 6.1 控制策略对比

| 策略 | 说明 | 论文用途 |
| --- | --- | --- |
| no-TES baseline | 不使用 TES，由即时冷却满足负荷 | 衡量 TES 和控制器是否带来增量价值 |
| rule-based TES | 固定低价/高 PV 时段充冷，高价时段放冷 | 衡量 MPC 是否优于简单规则 |
| MILP-MPC TES | 使用预测信息滚动优化 TES 和购电行为 | 本文提出方法 |

如果时间不足，最低可接受对比是 `no-TES baseline` 与 `MILP-MPC TES`；但为了论文完整性，建议保留 `rule-based TES`。

### 6.2 场景矩阵

| 场景 | 目的 | 备注 |
| --- | --- | --- |
| `nominal` | 验证基本控制行为 | 基准天气、PV 和电价 |
| `baseline_no_tes` | 无 TES 对照 | 成本、PUE、PV 消纳基线 |
| `rule_based_tes` | 简单 TES 规则对照 | 判断 MPC 的增量 |
| `pv_error_5` | 轻微 PV 预测误差 | 鲁棒性 |
| `pv_error_10` | 中等 PV 预测误差 | 鲁棒性 |
| `pv_error_20` | 较强 PV 预测误差 | 鲁棒性边界 |
| `hot_week` | 高温天气压力 | 温度和冷量约束压力 |
| `mild_week` | 温和天气条件 | TES 利用率变化 |
| `tariff_low` | 低电价价差 | 经济性下限 |
| `tariff_base` | 基准电价价差 | 标准情况 |
| `tariff_high` | 高电价价差 | TES 价值上限 |

### 6.3 评价指标

| 类别 | 指标 |
| --- | --- |
| 经济性 | total electricity cost, cost saving ratio, grid import, peak grid demand |
| PV 利用 | PV self-consumption ratio, PV load coverage ratio, PV spill |
| 温度与安全 | temperature violation count, violation degree-hours, max temperature, p95 temperature |
| TES 行为 | SOC trajectory, SOC violation count, equivalent cycles, charge/discharge energy |
| 求解性能 | average solve time, p95 solve time, optimal solve rate, infeasible count |

评价指标应服务于解释机制，而不是只追求“节能百分比”。如果实验结果显示成本改善有限，论文应讨论 PV 规模、TOU 价差、TES 效率、冷机 COP、终端 SOC 约束和温度约束等边界条件。

## 8. 风险与控制

| 风险 | 影响 | 控制办法 |
| --- | --- | --- |
| 当前仓库没有完整 MPC 控制层 | 无法直接跑论文实验 | 先实现最小控制层，不把旧代码恢复当作必要前提 |
| EnergyPlus 与 MPC 闭环耦合成本过高 | 进度失控 | 论文主线采用控制导向代理模型，EnergyPlus 作为背景模型和后续高保真扩展 |
| MPC 成本节省不明显 | 结论压力 | 将论文重点放在可解释约束调度和适用边界分析，不硬造正结果 |
| PV 规模或电价价差不足 | TES 价值不明显 | 用 tariff sensitivity 和 PV 预测误差分析解释条件依赖 |
| MILP 求解时间过长 | 闭环仿真慢 | 控制步长、预测窗口和二进制变量数量保持最小；必要时先跑短周期 |
| 温度代理模型过粗 | 结果物理可信度不足 | 明确假设，使用 EnergyPlus 输出或合理参数校准，并在局限性中说明 |
| 论文主线再次发散 | 文档难收敛 | 正文只保留 TES-PV-TOU MILP-MPC，旧 DRL/失败实验放附录或删去 |

## 9. 论文写作边界

正文应围绕以下链条展开：

```text
问题背景 -> 文献缺口 -> 系统建模 -> MILP-MPC 控制方法 -> 实验设计 -> 结果分析 -> 适用边界与结论
```

不建议放入正文主线的内容：

- 旧 DRL 训练细节；
- Sinergym 内部实现；
- 大量失败实验；
- agent 通信记录；
- 旧 dashboard；
- 外部项目复现过程；
- EnergyPlus 全部对象级建模细节。

可以保留在附录或背景说明中的内容：

- 前期曾考虑强化学习；
- 因样本效率、安全约束和部署风险，最终转向 MPC；
- EnergyPlus 模型作为高保真背景和后续闭环扩展基础。

## 10. 最小验收口径

本项目不以“节能百分比很高”作为唯一成功标准。最低可接受毕业设计成果应满足：

1. 有一个边界清楚的数据中心 TES-PV-TOU 控制问题定义。
2. 有一个数学形式完整、可求解的 MILP-MPC 控制器。
3. 有 no-TES baseline 和至少一个 TES-MPC 闭环场景对比。
4. 有成本、PV 消纳、温度约束、SOC 行为和求解性能指标。
5. 有对结果机制和适用边界的解释。
6. 论文中所有结论都能追溯到模型、代码、数据或实验输出。
