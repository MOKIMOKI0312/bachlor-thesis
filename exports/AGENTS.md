# exports/ 目录规则

## 用途

- 本目录只用于存放本地导出包、审查包、压缩包、临时交付物和给外部工具审阅的快照。
- 这些文件通常体积较大、可重复生成，并且可能包含某一时刻的运行结果副本。

## Git/GitHub 同步规则

- 默认不要把本目录中的导出文件、`.zip`、临时文件夹或审查包提交到 Git/GitHub。
- 本目录只允许跟踪版本管理规则文件本身：
  - `exports/AGENTS.md`
  - `exports/.gitignore`
- 如果用户明确要求把某个导出包入库，必须先说明风险，并只为该具体文件添加显式例外。
- 生成新的审查包后，可以在对话中给出本地路径，但不要自动 `git add` 这些导出内容。

## 操作要求

- 打包前优先确认源代码和结果目录已经在项目内保存。
- 打包文件名应包含主题和日期，例如 `chiller_tes_mpc_fixcheck_pro_review_20260505.zip`。
- 如果重新生成同名压缩包，可以覆盖本地文件；不要因此修改 Git 同步策略。
