## Context

M2-F1 阶段：MPC oracle / MPC-lite 已在 trainlike 672-step 通过 mechanism gate（`m2f1_mpc_oracle_672` / `m2f1_mpc_lite_672` 双 PASS，详见 `analysis/m2f1_mpc_lite_feasibility_check_202605.md`）。

本路线聚焦"3 周内毕业设计交付"，在已有可达性证明基础上：

1. **方法升级**：LP → MILP，与 Zhu 2023/2024、Tarragona 2021 综述主流一致，消除 LP 连续松弛产生的伪 charge+discharge 同时执行解。
2. **鲁棒性证据**：在 7 月代表月上做 forecast noise 注入（perfect / ±10% Gaussian / persistence-h）三场景对比，量化 mechanism gate 在预测扰动下的衰减。Zhu 2024 DMPC 已识别"参数不确定性会导致 mode 预测错误"，本项目用 forecast noise sweep 给出毕设级证据。
3. **基线 head-to-head**：rule baseline（valve=0，对应 E1-R2c 退化解）加入 MPC 矩阵，构成 3 控制器 × 3 forecast 场景 = 9 组主实验 + 1 组 official_ood `ITE_Set=1.0` stress test。
4. **论文 + 答辩 + handoff** 一并产出。

**不在范围**：BC/IL warm-start、全年评估、多站点扩展、stochastic MPC（chance constraint / scenario tree）。这些列入 limitations / future work。

## Week 1（Day 1–7）：MILP 升级 + 接口固化

### 目标
- MILP 求解器在 7 月代表周（672 step）上跑通，gate 与现有 LP-based 672-step PASS 一致或更好。
- forecast noise 三种 mode 接口 + 单元测试 + monitor schema 版本化。

### 任务

| ID | 任务 | 引用文献 / 现有代码 | 验收 |
|---|---|---|---|
| W1-1 | MILP 模型公式化：`v ∈ [-1,1]` 连续 + `b_charge, b_discharge ∈ {0,1}` binary + `b_charge + b_discharge ≤ 1` 互斥 + `v ≤ b_discharge - b_charge·δ_max·...`（big-M 链接）；目标函数与 §3.2 一致 | Zhu 2023 §5、Zhu 2024 §3、Jeong 2024 §"discrete vs continuous" | 公式写入 docstring，可在 PDF 引用 |
| W1-2 | 实现 `_build_milp(self, ...)` 替代 `_build_lp(...)`；`scipy.optimize.milp` 调用 | scipy 1.11+ HiGHS MILP | 单 step 求解 < 1 s |
| W1-3 | 单元测试：手工构造 24 step 小问题，验证 (a) low-price 段不放冷、(b) high-price 段不充冷、(c) charge_discharge 互斥、(d) infeasible 时 fallback heuristic | `pytest` 新文件 `tests/test_mpc_milp.py` | 4 个测试全 PASS |
| W1-4 | `--forecast-mode` 参数 + noise injection layer（perfect / Gaussian σ=0.10 / persistence-h）。Persistence-h：未来 24 h price/PV 全用 t 时刻当前值；Gaussian：真值 ×(1+ε), ε∼N(0, σ²) 逐点独立、固定 seed 可复现 | Gao 2025 §"实测误差 ±5%"、Tarragona 2021 §"forecast horizon" | smoke 跑 96 step 三种 mode 各 1 次，monitor 列 `forecast_mode` / `forecast_seed` / `forecast_sigma` 全部写入 |
| W1-5 | monitor schema 版本化：在 `result.json` 顶部加 `schema_version: 2`，把新增列（mode binary、forecast diagnostics）注释清楚 | Sinergym 2025 §"benchmark clarity" | 旧 v1 monitor 仍可读，对比表脚本 detect schema |
| W1-6 | 7 月代表周（trainlike `DRL_DC_training.epJSON`）672 step 重跑：oracle MILP perfect-forecast，验证 gate 与原 LP 结果一致或更好 | 现有 `m2f1_mpc_oracle_672` 作为 baseline | charge_window_sign_rate ≥ 0.80 / Δ_SOC_prepeak > 0 / Δ_SOC_peak < 0 |
| W1-7 | 写 W1 完成 handoff 到 `analysis/m2f1_w1_milp_handoff_20260510.md` | — | 含 git commit hash + 单元测试输出 + 7 月 672 step 数值 |

### 风险

- **MILP 求解时间**：24 h horizon × 96 step × binary 1×96 ≈ 全过程 2688 个 step 的 MILP，每步约 0.5–1 s，全月约 15–45 min。如果 > 1 h，启动 warm-start（前一步解作为 hint）；仍超时则把 MILP 降为"每 N 步求解一次 + 中间步沿用"近似策略，并在论文中报告。
- **HiGHS MILP API 差异**：`scipy.optimize.milp` 接口与 `linprog` 不同，integrality 参数易错。W1-3 单元测试是早期防线。

### 鲁棒性结论模板（提前固化叙事，避免 W3 写论文时反复改）

> "在 perfect forecast 下 MPC oracle/lite 双双 PASS mechanism gate（charge_window_sign_rate=1.0、Δ_SOC_prepeak=+0.25、Δ_SOC_peak=−0.18）；在 ±10% Gaussian forecast noise 下，charge_window_sign_rate 衰减到 X.XX，Δ_SOC_prepeak 衰减到 Y.YY；persistence-h forecast 下进一步降到 Z.ZZ。该衰减曲线表明 forecast 质量是 MPC 主线的关键依赖，与 Zhu 2024 DMPC §"参数不确定性 → mode prediction error"一致。论文不在本期建模 chance-constraint 缓解（列为 future work）。"

X/Y/Z 由 W2-2 汇总表决定；衰减解释 framing 现在固定。

### 风险

- **Persistence-h 下 gate 重度击穿**（charge_window_sign_rate < 0.4）：仍按用户决策"只报告衰减不修复"。叙事调整为"persistence forecast 表现 mechanism 不可达，证明 forecast quality 不可省"。
- **stress test (`ITE_Set=1.0`) MILP 解 infeasible**：fallback heuristic 接管；result.json 显式记录 fallback 比例。Wang 2022 ICCPS §"safety-rectification" 的 fail-loud 原则保留。
