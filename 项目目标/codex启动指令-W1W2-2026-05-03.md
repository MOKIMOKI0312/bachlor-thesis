### 4.1 W1-1：MILP 公式化（先纸笔写、再 commit docstring）

变量布局（必须严格按这个，方便后续单元测试和论文公式引用）：

- `v[k] ∈ [-1, 1]`：每步阀门目标（连续）。`k = 0..N-1`，`N = horizon_steps = 96`（24 h × 4 step/h）。
- `b_charge[k] ∈ {0,1}`：充冷模式 binary（`v[k] < 0` 时 1）。
- `b_discharge[k] ∈ {0,1}`：放冷模式 binary（`v[k] > 0` 时 1）。
- 互斥：`b_charge[k] + b_discharge[k] ≤ 1`。
- 链接（big-M）：`v[k] ≥ -b_charge[k]` 与 `v[k] ≤ b_discharge[k]`（M = 1，因为 `|v| ≤ 1`）。
- `soc[k] ∈ [0.05, 0.95]`：物理 SOC（连续，wrapper 严格 bounds 是 [0.10, 0.90]，差值是 LP 安全裕度）。
- `s_hi[k], s_lo[k] ≥ 0`：SOC 上下界 slack。
- `Δv` rate limit：`|v[k+1] - v[k]| ≤ δ_max = 0.20`。
- 模式约束：`b_discharge[k] = 0` 当 `is_low_price[k]`；`b_charge[k] = 0` 当 `is_high_price[k]`。
- low-price pre-peak 强制充冷：`v[k] ≤ -ε_charge` 当 `is_low_pre_peak[k] and soc[k] < target_high`（ε_charge = 0.05；具体公式参考现有 LP-based heuristic）。
- 目标函数：`Σ [c_TOU(k) · p_net(k) + λ_term · |soc[N-1] - soc_target| + λ_sched · |soc[k] - soc_schedule(k)| + λ_slack · (s_hi[k] + s_lo[k])]`，`λ_slack` 取 1e6 级别保证 slack 非零时 fail-loud 可见。

写到 `plan_tes_action_milp(...)` 的 docstring 里，要能直接复制到论文 Ch 3。

### 4.4 W1-4：forecast noise 注入

加 CLI：`--forecast-mode {perfect, gaussian, persistence}`，`--noise-sigma 0.10`，`--noise-seed 42`。

- `perfect`：MPC 直接读未来真值（现有行为）。
- `gaussian`：未来 24 h 真值 × `(1 + ε)`，`ε ~ N(0, σ²)` 逐点独立、`np.random.RandomState(seed)` 固定。**仅扰动 price 与 PV，不扰动 IT load**（IT load 是物理仿真直读不属于"预测"）。
